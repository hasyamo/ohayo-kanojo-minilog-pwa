# ミニログPWA「運営メモ」欄追加 指示書

## 概要

ミニログPWAに「運営メモ」テキスト入力欄を追加する。
note-stats-trackerダッシュボードで自動生成された「今日の状態メモ」を貼り付けるための欄。

## 対象リポジトリ

おはようカノジョのミニログPWA（GitHub Pagesで公開中）

## 追加する欄の仕様

### 位置
- 「理由 / 引っかかり」欄の下、「タグ」欄の上

### UI
- ラベル: `運営メモ（任意）`
- テキストエリア（複数行、3行程度の高さ）
- placeholder: `例：リーチ↑ アクション↑ スキ率→ | 好調日。`
- 任意入力（必須ではない）

### デザイン
- 既存の「判断」「理由 / 引っかかり」と同じスタイルを踏襲
- ダークテーマに合わせる

### Markdownコピー出力への反映
- 「Markdownをコピー」ボタン押下時の出力に含める
- 運営メモが空の場合は出力に含めない
- 出力フォーマット例：

```markdown
## ミニログ 2026-02-10（火）

**モード**: 観測
**判断**: 週末実装は触らず様子を見る
**理由**: コピペ負荷が高くてログが止まりがち
**運営メモ**: リーチ↑ アクション↑ スキ率→ | 好調日。
**タグ**: #観測 #運用
```

### プレーンコピー出力への反映
- 「プレーンをコピー」ボタン押下時も同様に含める
- 空の場合は出力に含めない

### localStorage
- 既存のlocalStorageスキーマに `commentary` フィールドを追加
- クリアボタン押下時にリセットされること

## 注意事項
- 既存のフォーム要素（モード、判断、理由、タグ）の動作を壊さないこと
- PWAのオフライン動作に影響しないこと
- 既存のスタイル・カラーテーマと統一すること
